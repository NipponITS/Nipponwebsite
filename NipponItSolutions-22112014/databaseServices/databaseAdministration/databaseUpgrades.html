Database Upgrades

We @ NipponITS believe in seamless upgrades without potential impact to business during and after Migration.

With Fast moving Technology and changes in Database Products there are lot of new Versions are released. It is very important to be in pace with Technology. Expert DBAs of Nippon ITS will provide support in planning, Designing and upgrading you infrastructure.

Following is the Upgrade process we Follow @ NipponITS

  • Consulting for Migration Database Analysis.
    This consulting phase we will analyze the database to be upgraded and check if existing infrastructure is good enough as per new version. This analysis will help in deciding migration Path for the database. We also provide support to application team to understand what will change as part of new version so they can analyze impact on Application.
  • Preparing the Documentation and Analysis of migration (Cook Book).
  • Presentation of the Migration Actions to be performed and possible issues that might be encountered during and post migration. This presentation also indicating new features of new version , its impact based on performance, availability ,reliability and Recoverability.
  • Rollback process document. (Rollback Cook book)
  • Decision made on method of migrations and Downtime requirements along with Customer.
  • On Customer approval of the migration effects and timelines from Customer Migration Execution is initiated.
  • Final Report and presentation of migration which contains end to end migration details and differences if any.
  • Post Upgrade support for 10 days.

NipponITS is capable to work on Upgrade of following products :

  • Oracle Database All versions to oracle 11gR2 or Oracle 12c.
  • Microsoft SQL server all version to 2012 R2, DTS to SSIS.
  • MYSQL all version to 5.x Upgrade
  • Oracle Applications EBS Database All version to EBS 12.
