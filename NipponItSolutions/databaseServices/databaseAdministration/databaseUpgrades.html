Database Upgrades

We @ NipponITS believe in seamless upgrades without potential impact to business during and after Upgrade.

With Fast moving Technology and changes in Database Products there are lot of new Versions are released. It is very important to be in pace with Technology. Expert DBAs of Nippon ITS will provide support in planning, Designing and upgrading the database infrastructure.

Following is the Upgrade process we Follow @ NipponITS

  • Consulting for Database Upgrade Analysis.
    During this consulting phase we will analyze the database to be upgraded and check if existing infrastructure is good enough as per new version. This analysis will help in deciding Upgrade Path for the database. We also provide support to application team to understand what will change as part of new version so they can analyze impact on Application.
  • Preparing the Documentation and Analysis of Database Upgrade (Cook Book).
  • Presentation of the Database Upgrade Actions to be performed and possible issues that might be encountered during and post Upgrade. This presentation also indicating new features of new version and what might be the impact based on Performance, Availability, Reliability and Recoverability of the Database system.
  • Rollback process document. (Rollback Cook book)
  • Decision is made on method of Database Upgrade and Downtime requirements along with Customer.
  • On Customer approval of the Upgrade effects and timelines; Database Upgrade Execution is initiated.
  • Final Report and presentation of Database Upgrade which contains end to end Upgrade details and differences if any.
  • Post Database Upgrade support for Agreed period.

NipponITS is capable to work on Upgrade of following products

  • Oracle Database all versions to oracle 11gR2 or Oracle 12c.
  • Microsoft SQL server all version to 2012 R2, DTS to SSIS.
  • MYSQL all version to 5.x Upgrade.
  • Oracle Applications EBS Database all version to EBS 12.
